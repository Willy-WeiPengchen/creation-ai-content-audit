# 贡献规则（Contributing Guidelines）
本项目欢迎开发者针对开源基础功能进行贡献，所有贡献需符合 Apache License 2.0 协议及以下规则，确保项目合规、有序迭代，实现社区自助互助、自主运转。

## 一、贡献范围
1.  仅接受针对开源基础功能的 bug 修复、接口优化、文档完善（如基础收录、基础核验、基础检索相关）；
2.  允许提交基础接口的扩展代码、基础配置优化，需符合项目基础架构，不涉及未开源功能及商业闭环载体；
3.  禁止提交涉及未开源功能（智能确权、分润结算、防侵权核心算法等）、商业闭环载体的代码或逻辑；
4.  欢迎社区开发者自发贡献 Wiki 文档、使用 FAQ、Docker 镜像等实用资源，助力社区自助互助。

## 二、贡献要求
1.  提交的代码需符合行业规范，结构清晰、注释完整，无冗余代码、无安全漏洞；
2.  贡献者需保证提交的代码为原创或已获得合法授权，不侵犯第三方知识产权、商业秘密；
3.  提交代码前需自行测试，确保不影响开源版现有功能的正常运行；
4.  修改现有代码时，需在文件头部显著标注修改日期、修改人及变更内容，同时保留标准Apache 2.0源码头部声明；
5.  文档类贡献（如 README 优化、使用说明补充、FAQ 完善）需保证内容准确，与项目实际功能一致，便于社区开发者参考使用。

## 三、贡献流程
1.  Fork 本项目至个人仓库；
2.  在个人仓库中创建新分支（命名格式：feature/优化功能名 或 fix/bug 描述）；
3.  完成代码编写或文档修改后，提交 Commit（清晰描述修改内容）；
4.  提交 Pull Request 至本项目 main 分支，在描述中清晰说明：
    - 修改目的与背景
    - 具体变更内容
    - 本地测试结果或验证方式
5.  项目维护者审核通过后，合并至主分支；若审核未通过，需根据反馈修改后重新提交；
6.  活跃贡献者可申请“Community Maintainer”角色（无代码合并权，仅可协助管理 Issue、引导社区讨论）。

## 四、知识产权说明
1.  所有提交的贡献代码，均根据 Apache License 2.0 提供许可；
2.  贡献者提交代码即视为同意：其贡献受 Apache License 2.0 约束，且明确知悉本项目的核心专利（申请号 202610230541.6）及未开源模块（如智能确权、价值分配等）的权利仍归属权利人韦朋辰（Pengchen Wei），不因贡献行为而转移；
3.  贡献者保留其贡献代码的版权，但不得主张超出 Apache 2.0 协议的额外权利，不得通过贡献代码推导未开源功能。

## 五、禁止行为
1.  提交涉及未开源功能、商业闭环载体的代码或逻辑；
2.  提交抄袭、侵权、存在安全隐患的代码；
3.  恶意提交无效代码、冗余代码，干扰项目迭代或社区秩序；
4.  修改或删除项目中的版权声明、NOTICE文件内容、Apache 2.0 许可证信息；
5.  试图通过贡献代码推导、复现未开源功能，或复制、模仿商业闭环载体相关组件；
6.  在 Issue 区提交售后请求（如“无法部署”“求教程”等），干扰社区自助讨论氛围。
